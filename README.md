# Beleaf Render Service v3.1

Thai typography image renderer for the Beleaf n8n content pipeline.

## Critical fix

The renderer now resolves layout in this order:

1. `renderDirectives.structureType`
2. `creativeProfile.layout.structureType`
3. top-level `structureType`
4. legacy `design.designTemplate.layout`
5. legacy `design["Layout Preset"]`
6. `banner_top`

This prevents legacy `hero` values in `16_DESIGN_BIBLE` from overriding the new Creative Profile generated by WF2/WF3.

## Supported layouts

`banner_top`, `hero`, `floating`, `banner_left`, `corner`, `ribbon`, `diagonal`, `bottom_overlay`, `magazine`, `clinic`, `lifestyle`, `review`, `promotion`, `editorial`.

## Health check

`GET /health`

Expected version:

```json
{"ok":true,"version":"3.1.0-creative-profile-priority-fix"}
```

## Diagnostics

Every PNG response includes headers:

- `X-Beleaf-Render-Version`
- `X-Beleaf-Resolved-Layout`
- `X-Beleaf-Creative-Profile`

For P011, the resolved layout must be `floating` and profile layout ID should be `L007`.
